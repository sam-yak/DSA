## ЁЯУП The Rules

1. **20-minute rule:** Struggle for 20 min, then study the solution. Then close it and reimplement from scratch.
2. **Always state complexity:** Time and space, out loud, every single problem.
3. **Code on plain text 30% of the time.** Google interviews use Google Docs.
4. **Narrate while solving.** Silent coding = red flag at Google.
5. **Track honestly.** тЬЕ = solved clean first try. тЪая╕П = needed hints. тЭМ = couldn't solve.
6. **Sleep 7+ hours.** Pattern recognition consolidates during sleep.

## ЁЯОд Google Interview Communication Framework

Use this on EVERY problem:

1. **CLARIFY** (1-2 min) тАФ Restate problem. Ask about edge cases, constraints, input size.
2. **PLAN** (3-5 min) тАФ Discuss approach. State time/space complexity BEFORE coding.
3. **CODE** (15-20 min) тАФ Clean, modular code. Descriptive variable names.
4. **TEST** (3-5 min) тАФ Walk through examples manually. Test edge cases.
5. **OPTIMIZE** (if time) тАФ Can we do better? Discuss tradeoffs.
