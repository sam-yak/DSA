## 🧠 The 4 Phases

### Phase 1: Reactivation (Days 1–5)
Rebuild core data structures and fundamental problem-solving patterns. Go from "can't solve Easy" to "confidently cracking Medium."

### Phase 2: Pattern Mastery (Days 6–13)
Deep-dive into the 12 essential patterns Google tests. One pattern per day. Memorize the template, then drill it through 4-5 problems.

### Phase 3: Integration & Hardening (Days 14–18)
Mixed Google-tagged problems under timed conditions. Build pattern recognition reflexes. Tackle Hard problems.

### Phase 4: Battle Simulation (Days 19–21)
Full mock interviews. Behavioral prep. Speed runs for confidence. Lock in everything.

## 📏 The Rules

1. **20-minute rule:** Struggle for 20 min, then study the solution. Then close it and reimplement from scratch.
2. **Always state complexity:** Time and space, out loud, every single problem.
3. **Code on plain text 30% of the time.** Google interviews use Google Docs.
4. **Narrate while solving.** Silent coding = red flag at Google.
5. **Track honestly.** ✅ = solved clean first try. ⚠️ = needed hints. ❌ = couldn't solve.
6. **Sleep 7+ hours.** Pattern recognition consolidates during sleep.

## 🎤 Google Interview Communication Framework

Use this on EVERY problem:

1. **CLARIFY** (1-2 min) — Restate problem. Ask about edge cases, constraints, input size.
2. **PLAN** (3-5 min) — Discuss approach. State time/space complexity BEFORE coding.
3. **CODE** (15-20 min) — Clean, modular code. Descriptive variable names.
4. **TEST** (3-5 min) — Walk through examples manually. Test edge cases.
5. **OPTIMIZE** (if time) — Can we do better? Discuss tradeoffs.
